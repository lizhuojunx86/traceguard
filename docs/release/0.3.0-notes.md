## PR description — copy into the GitHub PR

**Title:** `feat(traceguard): 0.3.0 — positioning, OTel export, contamination & loop groundwork`

**Body:**

> ### Summary
>
> 0.3.0 is a **positioning, evidence, and interoperability** round for the
> `traceguard` SDK. It is **purely additive** — the 0.2.0 public API, table
> schemas, and the binding SPEC clauses (§§3–5) are unchanged, so this is a
> SemVer **minor** bump. Heavy/optional capabilities ship behind extras and are
> never imported at package load.
>
> ### What's new
>
> - **OpenTelemetry / OpenInference export** behind `traceguard[otel]`
>   (`traceguard.exporters.otel`): `export_trace` / `export_traces` /
>   `trace_to_attributes` map a trace to an OTLP span carrying the
>   time-integrity attributes (`traceguard.input_hash`, `gen_ai.request.model` +
>   `traceguard.model.available_to_us_at`, `traceguard.prompt_template_hash`,
>   `traceguard.feature_as_of`, `openinference.span.kind`). The SQLite/SQLAlchemy
>   store stays the source of truth; OTel is an *additional* export (SPEC §6.1 /
>   §6.6).
> - **Training-contamination groundwork** in `traceguard.contamination`
>   (look-ahead *kind 1*): `min_k_prob` (MIN-K% PROB membership-inference
>   baseline), `performance_decay_across_regimes`, the `ClaimVerifier` protocol
>   + `ClaimVerdict`, and `attach_contamination_score` (records scores via a
>   trace's `output_parsed` JSON — no schema change). The
>   `traceguard[contamination]` extra reserves the dependency-isolation point for
>   heavier future work and is currently empty (baselines are standard-library).
> - **Loop evidence-gating** in `traceguard.loop`: `EvidenceGate` and the
>   `evidence_gated` decorator admit a memory write only if its evidence is
>   sourced at/before a cutoff — the loop-level companion to invariant 1.
> - **Docs:** `docs/POSITIONING.md` (two kinds of look-ahead, anti-positioning,
>   research anchors), `docs/loop-integration.md`, and sharpened READMEs / SPEC
>   positioning preambles (all non-normative; §§3–5 unchanged).
> - **Examples:** `model_anachronism.py`, `prompt_drift.py` (runnable), plus
>   `training_contamination.py` and `loop_self_contamination.py` (run a real
>   slice, sketch the rest), and `examples/README.md`.
>
> ### Compatibility
>
> - **No breaking changes.** Only existing source file modified is the top-level
>   `__init__.py` (docstring + `__version__`); `__all__` is byte-identical. The
>   three new subpackages are reachable only via explicit
>   `from traceguard.<subpkg> import …` and are not re-exported at the top level.
> - **No new heavy import at load time** — verified `import traceguard` (and even
>   importing all three new subpackages) pulls zero `opentelemetry` / `anthropic`
>   modules; the OTel dep is isolated inside `traceguard.exporters.otel`.
> - **SPEC:** binding clauses §§3–5 unchanged (47 insertions / 0 deletions across
>   both SPEC files; all additions explicitly non-normative — §0/§1 positioning,
>   §6.1/§6.6 opt-in extensions).
> - **Downstream:** `quant_alpha_v2` pins `v0.2.0-phase0` and is unaffected
>   (additive).
>
> ### Verification
>
> - `uv run pytest` → **70 passed** (44 baseline + 26 new across
>   `test_contamination.py` / `test_exporters_otel.py` / `test_loop.py`).
> - `ruff check .` → clean.
> - `python -m build` → sdist + wheel; `twine check` → both **PASSED**.
> - All three extras resolve against PyPI (`[otel]` → opentelemetry-api/sdk;
>   `[anthropic]` → anthropic; `[contamination]` → core only).
>
> ### Citations & caveats
>
> - Research-anchor arXiv citations in the READMEs / `docs/POSITIONING.md` are
>   **verified** (2512.23847, 2601.13770, 2602.17234; MIN-K% PROB 2310.16789).
> - The flagship case study (`docs/case-studies/fmp-revision.md`) is kept
>   **local-only** (its directory is `.gitignore`d as a real-data guardrail) and
>   lives outside the SDK package boundary, so it never ships in the wheel/sdist.
>   Its headline numbers were reconciled against the quant_alpha_v2 vintage
>   harness on 2026-06-16 (41.4% / 15.3% / four-month window bit-reproduced; the
>   ~73% / ~82% retention pair is as-of the 2026-06-05 run, FINAL leg drifts);
>   its arXiv citations are verified too (2512.23847 / 2601.13770 / 2602.17234 /
>   2310.16789).

## GitHub release note — copy into the `v0.3.0` release

**Tag:** `v0.3.0`  **Title:** `traceguard 0.3.0`

> **Positioning, evidence, and interoperability — purely additive (SemVer minor).**
> The 0.2.0 public API and the binding SPEC clauses are unchanged.
>
> **Added**
> - `traceguard[otel]` — export traces as OpenTelemetry / OpenInference (OTLP)
>   spans carrying time-integrity attributes; the SQLite/SQLAlchemy store stays
>   the source of truth.
> - `traceguard.contamination` — training-contamination groundwork (look-ahead
>   kind 1): `min_k_prob`, `performance_decay_across_regimes`, `ClaimVerifier`
>   protocol, `attach_contamination_score` (via `output_parsed`, no schema
>   change). Detection only.
> - `traceguard.loop` — `EvidenceGate` + `evidence_gated`: admit a loop memory
>   write only if its evidence is sourced before a cutoff.
> - Docs: `docs/POSITIONING.md` (two kinds of look-ahead), `docs/loop-integration.md`;
>   four new examples.
>
> **Changed**
> - READMEs and SPEC positioning sharpened (time-integrity layer; two kinds of
>   look-ahead; anti-positioning vs Langfuse/Phoenix/LangSmith/Helicone; research
>   anchors). Contract clauses (SPEC §§3–5) unchanged; §6.1/§6.6 register the
>   opt-in extensions.
>
> **Notes**
> - Research anchors are verified citations: arXiv 2512.23847, 2601.13770,
>   2602.17234, and MIN-K% PROB 2310.16789.
> - The `fmp-revision` case study is kept local (real-data guardrail); its
>   headline numbers are reconciled (41.4% / 15.3% / four-month window verified;
>   ~73% / ~82% as-of the 2026-06-05 run).
>
> **Install:** `pip install traceguard` · extras: `traceguard[otel]`,
> `traceguard[anthropic]` · Python 3.11+ · Apache-2.0
>
